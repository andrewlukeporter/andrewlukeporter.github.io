University Projects

Queen's University Belfast - 2013/17



In level 2, in the Android module as part of a team of 3, I got the opportunity to develop an Android game of my choice.

I decided that we should develop a space invaders style game. There were several challenging issues in this module, as all of the other members of my team dropped out of the module, resulting in me developing the entire game. After every sprint we would get feedback on our progress in the module however this would be feedback if 3 people had developed what had been done so far and so was not accurate.

An agile approach was taken towards this project.

In level 2, in the Software Engineering module, I got the opportunity to develop a monopoly style game using Java (Swing) as part of a team of 11.

Within this module, my roles within the project were analysing what screens were needed, designing the images, and testing the system using JUnit.

The reason I took a testing role within this project was because I was I hadnt tested using JUnit before and felt that it would be more beneficial for me to learn to see if it would be something I was interested in

A spiral approach was taken towards this project.

In level 1, in the fundamentals of programming module I got the opportunity to design and implement a tamagotchi pet game using Swing (Java). In this game, there was a pet selection screen which was to be implemented as part of a team of 6, and then there was an interface for each of the 6 different pets.

In the programming challenges module, I got to extend this tamagotchi game. The functionality I added was the ability to receive coins and the ability to go to the shop and purchase items with them.

A waterfall approach was taken to the module.

In level 1, in the software engineering module I got the opportunity to develop a ticketing website for an event called 'The Big Event' which was going to be held in Belfast in 2015.

The website was initially developed with Microsoft Visual Studio and used ASP.NET MVC 3 (C#).

However, we changed to HTML, PHP, CSS and Javascript after several members of the team struggled to develop in C#

I decided that we should take an iterative approach to the project. We got feedback on progress from the client from Deloitte at 4 different points during the module.

I developed a prototype system, which had all of the pdfs stored in the cloud, there was a download pdfs button and 2 buttons for 2 different topics, however, my neighbour didnt want the users to be able to make copies of the content, and my app would not prevent users from making copies, in fact it would actually store the files in a folder on their device.

In the end the app was not used and the pdfs are due to be published on the kindle bookstore.
